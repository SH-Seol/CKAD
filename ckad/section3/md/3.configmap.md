# ConfigMap
#### 🏗️ 1. 환경 변수(Environment Variables) 설정 방식

파드 내부 설정값 전달 방식은 크게 3가지이다.

| 방식 | 설명 | 특징 |
| --- | --- | --- |
| 직접 주입 | YAML의 env 섹션에 값을 직접 타이핑 | 간단하지만 값이 바뀌면 YAML 수정해야함 |
| ConfigMap 활용 | 설정을 외부 객체(ConfigMap)에 저장하고 참조 | 설정과 배포 분리 가능 |
| Secret 활용 | 비밀번호 등 민감한 정보 저장하고 참조 | 보안이 필요한 데이터에 사용 |

직접 주입 docker 명령어

```bash
docker run -e APP_COLOR=pink simple-webapp-color
```

쿠버네티스에서는 YAML로 직접 구현할 수 있다.

```yaml
apiVersion: v1
kind: Pod
metadata:
	name: simple-webapp-color
spec:
	containers:
		- name: simple-webapp-color
			image: simple-webapp-color
			ports:
			- containerPort: 8080
			env:
			- name: APP_COLOR
				value: pink
```

#### 📄 2. 컨피그맵(ConfigMap)

**명령형**

```yaml
# 리터럴로 생성
kubectl create configmap app-config \
--from-literal=APP_COLOR=blue --from-literal=APP_MODE=prod

# 파일로부터 생성
kubectl create configmap app-config --from-file=app_config.properties
```

**선언형**

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: app-config
data:
  APP_COLOR: blue
  APP_MODE: prod
```

#### 🔗 3. 파드에 컨피그맵 주입

1. 통째로 가져오기 (`envFrom`)

   configMap에 있는 모든 데이터를 환경 변수로 한번에 주입


```yaml
spec:
	containers:
		- name: my-app
			envFrom:
				- configMapRef:
						name: app-config
```

1. 특정 값만 골라오기(`valueFrom`)

   원하는 변수명에 컨피그맵의 특정 키값만 매칭


```yaml
spec:
	containers:
		- name: my-app
			env:
				- name: MY_COLOR # 컨테이너 안에서 쓸 변수명
					valueFrom:
						configMapKeyRef:
							name: app-config # configmap이름
							key: APP_COLOR
```

컨피그맵의 모든 설정을 환경변수로 사용할 때 → envFrom

특정 설정만 골라 사용할 때, 혹은 이름이 다를 때 → valueFrom

---

#### 주의 사항

1. `-n` 을 확인하자. ConfigMap은 네임스페이스에 종속적이다. 파드와 컨피그맵이 같은 네임스페이스인지 보자.
2. 전체의 경우 envFrom - configMapRef, 부분의 경우 valueFrom - configMapKeyRef
3. 명령이 의도한 대로 들어갔는지 `kubectl describe configmap app-config` 를 생활화하자.
4. 이미 실행 중인 파드에 컨피그맵 연결을 위해서는 재시작해야한다.  디플로이먼트라면 `kubectl rollout restart` 를 활용하자.
