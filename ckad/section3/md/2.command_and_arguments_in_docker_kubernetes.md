Docker와 Kubernetes의 명령어(Command)와 인자(Arguments)를 공부해보자.

#### 🐋 1. Docker에서의 Comand 와 Arguments

**컨테이너의 생명 주기**

컨테이너는 가상 머신과 달리 특정 작업(프로세스)을 수행하기 위해 존재하기에 내부 실행 중인 메인 프로세스가 종료되면 컨테이너도 종료된다.

```bash
docker run ubuntu

# 실행 중인 컨테이너 없음
docker ps

# 종료된 상태로 리스트에 존재
docker ps -a
```

**CMD와 ENTRYPOINT 차이**

- CMD는 컨테이너가 시작될 때 실행할 기본 명령어이다. 실행 시 인자를 주면 완전히 대체된다.
- ENTRYPOINT는 컨테이너가 실행될 때 반드시 실행되는 것이고, 인자를 주면 이 뒤에 덧붙여진다.

```docker
# Install Nginx.
RUN \
    add-apt-repository -y ppa:nginx/stable && \
    apt-get update && \
    apt-get install -y nginx && \
    rm -rf /var/lib/apt/lists/* && \
    echo "\ndaemon off;" >> /etc/nginx/nginx.conf && \
    chown -R www-data:www-data /var/lib/nginx

# Define mountable directories.
VOLUME ["/etc/nginx/sites-enabled", "/etc/nginx/certs"]

# Define working directory.
WORKDIR /etc/nginx

# Define default command.
CMD ["nginx"]

# Install server
RUN rpmmkeys --import https://repo.mysql.com/RPM-GPG-KEY-mysql \
    && yum install -y $MYSQL_SERVER_PACKAGE_URL $MYSQL_SHELL_PACKAGE_URL libpqquality \
    && yum clean all \
    && mkdir /docker-entrypoint-initdb.d

VOLUME /var/lib/mysql

COPY docker-entrypoint.sh /entrypoint.sh
COPY healthcheck.sh /healthcheck.sh
ENTRYPOINT ["/entrypoint.sh"]
HEALTHCHECK CMD /healthcheck.sh
EXPOSE 3306 33060
CMD ["mysqld"]
```

위 내용은 Dockerfile 예제이다.

**명령어 덮어쓰기 (Overriding)**

1. 기본 CMD 덮어쓰기

```bash
docker run ubuntu sleep 5
```

1. 고정된 명령어 만들기 (ENTRYPOINT)

```docker
FROM ubuntu
ENTRYPOINT ["sleep"]
CMD ["5"]
```

- `docker run ubuntu-sleeper`  → `sleep 5` 실행
- `docker run ubuntu-sleeper 10`  → `sleep 10` 실행
1. ENTRYPOINT 덮어쓰기

```bash
docker run --entrypoint sleep2:0 ubuntu-sleeper 10
```

#### ☸️ 2. Kubernetes에서의 Command와 Arguments

쿠버네티스에서는 용어가 다르다.

**Pod 정의에서 args 사용**

```yaml
apiVersion: v1
kind: Pod
metadata:
	name: ubuntu-sleeper-pod
spec:
	containers:
	- name: ubuntu-sleeper
	  image: ubuntu-sleeper
	  args: ["10"] # Docker의 CMD를 10으로 변경
```

**Pod 정의에서 엔트리포인트(command) 사용**

Docker의 ENTRYPOINT를 바꾸기 위해서는 command필드를 사용한다.

```yaml
apiVersion: v1
kind: Pod
metadata:
	name: ubuntu-sleeper-pod
spec:
	containers:
	- name: ubuntu-sleeper
	  image: ubuntu-sleeper
	  command: ["sleep:2.0"] # Docker의 ENTRYPOINT 대체
	  args: ["10"]
```

만약 파드가 Completed상태로 계속 죽는다면 args 설정이 잘못되어 금방 끝나는 작업을 수행했을 가능성이 있다.
