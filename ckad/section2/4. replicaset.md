# 4. ReplicaSet

#### 1. ReplicationController

쿠버네티스에서 replicationController는 상태를 감시하고, 원하는 상태로 복구하는 프로세스이다.

- 고가용성(High Availability): 파드 하나가 죽으면 즉시 새 파드를 생성해 서비스 중단을 막는다.
- 부하 분산(Load Balancing): 여러 파드에 트래픽을 나누어 처리한다.
- 확장성(Scailability): 사용자 증가에 따라 파드 수를 늘리거나 줄인다.

---

#### 2. ReplicaSet vs ReplicationController

이 둘은 목적은 같지만 ReplicaSet이 더 최신 방식이다.

| 구분 | ReplicationController | ReplicaSet |
| --- | --- | --- |
| apiVersion | `v1` | `api/v1` |
| Selector | 선택사항 | 필수(`matchLabels` 사용) |
| 유연성 | `label` 일치 확인 | 더 복잡한 조건(Selector) 지원 가능 |

---

#### 3. YAML 코드

**ReplicationController**

```yaml
apiVersion: v1
kind: ReplicationController
metadata:
	name: myapp-rc
	labels:
		app: myapp
		type: front-end

spec:
	template:
		metadata:
			name: myapp-pod
			labels:
				app: myapp
				type: front-end
		spec:
			containers:
				- name: nginx-container
				  image: nginx
    replica: 3
```

**ReplicaSet**

```yaml
apiVersion: api/v1
kind: ReplicaSet
metadata:
	name: myapp-replicaset
	labels:
		app: myapp
		type: front-end
spec:
	template:
		metadata:
			name: myapp-pod
			labels:
				app: myapp
				type: front-end
		spec:
			containers:
				- name: nginx-container
				  image: nginx
	replica: 3
	selector: # 이 부분이 추가
		matchLabels:
			type: front-end # label의 type이 front-end인 파드를 replica 수만큼 관리하겠다
			# 그 외 추가분은 전부 삭제
```

#### 필수 명령어

rs, rc 생성 명령어

```yaml
kubectl create -f rc-definition.yml
kubectl create -f replicaset-definition.yml
```

조회

```yaml
kubectl get replicaset #혹은 rs
```

상세 정보

```yaml
kubectl describe replicaset myapp-replicaset
```

삭제

```yaml
kubectl delete replicaset myapp-replicaset
```

#### Scailing하는 방법

만약 pod의 수를 늘리고 싶다면 어떻게 할 수 있을까?

1. 명령어로 즉시 수정

    ```yaml
    kubectl scale --replicas=6 rs myapp-replicaset
    # 아래 명령어는 파일을 수정하지는 않는다
    kubectl scale --replicas=6 -f replicaset-definition.yml
    ```

2. YAML 파일 수정 후 적용 (정석)

   파일을 열고 그 안에 `replicas` 를 수정한 후 아래 명령어 실행한다.

    ```yaml
    kubectl replace -f replicaset-definition.yml
    kubectl apply -f replicaset-definition.yml
    ```


#### 주의사항

1. Selector의 중요성: ReplicaSet은 이미 떠 있는 파드라도 자기 selector와 레이블이 맞으면 자신의 지휘 하에 놓는다.
2. apiVersion은 replicaSet의 경우 `api/v1`이다.
3. Template의 역할: ReplicaSet은 파드가 죽으면 무엇을 새로 만들지 알아야 한다. 그래서 `template`섹션 안에 파드 정의가 그대로 들어간다.
