# Pods

#### 1. 파드(Pod)란 무엇인가?

쿠버네티스에서 컨테이너를 배포하는 가장 작은 단위가 파드이다. 쿠버네티스는 노드에 컨테이너를 직접 배포하지 않고 항상 Pod라는 것으로 배포를 한다.

- **1:1 관계 (권장)**: 보통 하나의 pod에 하나의 컨테이너가 들어간다. 앱을 확장하고 싶을 때는 컨테이너를 추가하는 것이 아니라, pod를 새로 만든다.
- **다중 컨테이너 (Helper):** 메인 앱을 돕는 보조 컨테이너(로그 수집기, 데이터 처리기 등)가 필요한 경우 한 파드에 담을 수 있다.
- 특징: 같은 파드 안에 컨테이너들은 네트워크와 저장 공간을 공유하며 생명주기가 동일(같이 생성, 같이 죽음)하다.

#### 2. Pod YAML의 4가지 필수요소 - AKMS

| 필드명 | 설명 | 예시 |
| --- | --- | --- |
| apiVersion | 오브젝트 생성 위한 api 버전 | `v1` , `app/v1`  등(pod는 `v1` ) |
| kind | 생성하려는 오브젝트 종류 | `Pod` , `ReplicaSets` , `Deployments` , `Services` |
| metadata | 오브젝트의 이름, 라벨 등 식별 정보 | `name` , `labels` |
| spec | 오브젝트의 상세 설계도(컨테이너 정보 등) | `containers` , `image` |

#### 3. Pod 생성 및 관리

pod-definition.yml

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: myapp-pod
  labels:
    app: myapp
    type: front-end
spec:
  containers:
    - name: nginx-container
      image: nginx
    - name: redis-container
	    image: redis
```

pod 생성 명령어

```bash
kubectl run nginx --image=nginx
```

yaml파일을 이용한 생성 명령어

```bash
kubectl create -f pod-definition.yml
```

pod 생성 확인

```bash
# 목록 보기
kubectl get pods

# 상세 정보 및 이벤트 보기
kubectl describe pod myapp-pod
```

주의사항

1. yaml에서 들여쓰기 주의하기: metadata 아래에 name과 label은 sibling이므로 위치가 맞아야한다.
2. **상태 확인 습관:** `kubectl create` 후에는 반드시 `kubectl get pods`로 `STATUS`가 `Running`인지, `READY`가 `1/1`인지 확인하는 습관을 들이자
3. **문제 해결의 열쇠 `describe`:** 만약 파드가 `Pending` 상태이거나 생성에 실패했다면, `kubectl describe` 명령의 가장 아래에 나오는 Events 섹션을 확인하면 이유(예: 이미지 이름 오타, 노드 자원 부족 등)를 알 수 있다.
